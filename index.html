A/B Testing for Cacti E-Commerce

See how users shopped for cacti

A & B Pages

In order to collect data one of A or B websites were presented to the user randomly, and every interaction they had with the website was logged. Each log had a time stamp along with what the request was for. The main difference between A and B is the text dominance on the websites. A has much larger cacti descriptions and shows cart quantities on the homepage, while B has smaller text and user has to go to cart to see quantities added. Pictures of what A and B look like are given respectively.

Null Hypothesis

Since text is less memorable and there's less information on B, users will have to go back and forth homepage and cart more. Both return rates and time spent on the site should be higher for B than A

Alternative Hypothesis

If B isn't more visited that means the changes made either don't correlate with time to completion and return rates, or A turned out to keep people on the website longer than B.

Why We Should Use A & Key Takeaways

  • Users preferred stay on the website more while having more readable information on the page.
  • Regardless of the amount of information on the webpage, users want to click back and forth as little as possible.
  • If a user stays on a website for longer, they're more likely to buy things. Therefore keeping users on the website through a lot of text that is easy to read can be preferable.
  • There would be better statistical results with larger datasets, but using same number of data on both A and B for time to completion made it more accurate.
